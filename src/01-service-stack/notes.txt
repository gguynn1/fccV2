Required tools/dependencies:
- Node.js (LTS) with TypeScript (strict, ESM) for all runtime modules.
- `src/types.ts` for shared enums and cross-service vocabulary.
- Respect service and type boundaries: no direct runtime imports from sibling services.
- Local `types.ts` in this folder for scope-owned contracts.
- Stack-level contracts for transport, identity, classification, queueing, worker orchestration, and routing outcomes.
- Queue item and action result shapes that can move cleanly between services.

Recommended tools/dependencies:
- `tsc --noEmit` to verify type boundaries.
- `eslint` to keep imports and unused symbols clean.
- `prettier` to keep docs and source formatting consistent.
- `vitest` for fixture-driven verification where the scope has behavior worth testing.
- Scenario runners that validate the full top-to-bottom pipeline.
- Mocks for supporting-service calls while testing stack behavior.

Technologies and integrations:
- This stack coordinates Twilio transport, Anthropic Claude API classification/composition, BullMQ/Redis queueing, SQLite-backed state, and XState-driven escalation through service contracts.
- Inbound path: Twilio webhooks arrive via ngrok tunnel to Fastify on localhost:3000.
- Outbound path: direct HTTPS calls to Twilio REST API and Anthropic Claude API.
- Service boundaries stay explicit through TypeScript interfaces and discriminated unions.
- Vitest scenario coverage is the best fit for verifying end-to-end flow through the stack.
- Resilience: Redis AOF persistence required across the stack; stale catch-up logic on startup; BullMQ retry with exponential backoff on network failure.
