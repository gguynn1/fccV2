Required tools/dependencies:
- Node.js (LTS) with TypeScript (strict, ESM) for all runtime modules.
- `src/types.ts` for shared enums and cross-service vocabulary.
- Respect service and type boundaries: no direct runtime imports from sibling services.
- Local `types.ts` in this folder for scope-owned contracts.
- Topic and intent classification logic that reads content plus recent thread context.
- Access to recent conversation history so short replies can be interpreted in context.
- TopicKey-driven outputs that downstream services can trust.

Recommended tools/dependencies:
- `tsc --noEmit` to verify type boundaries.
- `eslint` to keep imports and unused symbols clean.
- `prettier` to keep docs and source formatting consistent.
- `vitest` for fixture-driven verification where the scope has behavior worth testing.
- Evaluation fixtures for ambiguous messages, forwarded content, and one-word replies.
- A repeatable classification benchmark set for regression checking.

Technologies and integrations:
- LLM provider: Anthropic Claude API for topic and intent classification.
- Classifier prompts should use recent thread context plus normalized transport/identity inputs.
- Structured outputs should be validated with Zod before the worker consumes them.
- Vitest benchmark fixtures are the safest way to catch classification regressions over time.
