Required tools/dependencies:
- Respect service boundaries: no direct runtime imports from sibling services.
- State-service write contracts for silent outcomes.
- Persistent record shapes for logs that should not trigger outbound messages.

Recommended tools/dependencies:
- Fixtures for silent records such as vendor history, pet care logs, and status entries.
- Queries that surface stored data only when explicitly requested.

Technologies and integrations:
- Write path: State service client (SQLite via better-sqlite3) — insert the record with no outbound side effect.
- Record types: Completed task logs, vendor job history, pet care entries, family status updates — each topic owns its record shape.
- Query surface: Records are only surfaced when a participant asks or when a digest explicitly includes them; no push notifications.
- Indexing: SQLite indexes on topic + entity + timestamp for efficient retrieval when records are queried.
- Testing: Vitest suites that write silent records and verify they do not trigger any transport or scheduler calls.
