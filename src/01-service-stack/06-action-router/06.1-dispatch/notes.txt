Required tools/dependencies:
- Node.js (LTS) with TypeScript (strict, ESM) for all runtime modules.
- `src/types.ts` for shared enums and cross-service vocabulary.
- Respect service and type boundaries: no direct runtime imports from sibling services.
- Transport-layer handoff contracts for immediate outbound delivery.
- Target-thread and payload validation before sending.

Recommended tools/dependencies:
- `tsc --noEmit` to verify type boundaries.
- `eslint` to keep imports and unused symbols clean.
- `prettier` to keep docs and source formatting consistent.
- `vitest` for fixture-driven verification where the scope has behavior worth testing.
- Immediate-priority test cases for urgent alerts and direct replies.
- Delivery-status checks for dispatched messages.

Technologies and integrations:
- Dispatch uses the Twilio REST API for immediate outbound phone-native messages.
- Outbound payloads should be validated with Zod before transport handoff.
- Twilio status callbacks plus pino logs are the right place to track delivery outcomes.
- Network outage: failed sends retry via BullMQ exponential backoff until internet returns; items queue locally, not dropped.
