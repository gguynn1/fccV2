Required tools/dependencies:
- Node.js (LTS) with TypeScript (strict, ESM) for all runtime modules.
- `src/types.ts` for shared enums and cross-service vocabulary.
- Respect service and type boundaries: no direct runtime imports from sibling services.
- Scheduler handoff contracts for batched items.
- Digest-window and quiet-window timing data for held outbound messages.

Recommended tools/dependencies:
- `tsc --noEmit` to verify type boundaries.
- `eslint` to keep imports and unused symbols clean.
- `prettier` to keep docs and source formatting consistent.
- `vitest` for fixture-driven verification where the scope has behavior worth testing.
- Time-based tests for morning digest, evening check-in, and daytime quiet behavior.
- Batch-merging fixtures that verify multiple items combine cleanly.

Technologies and integrations:
- Held items move into BullMQ scheduling backed by shared Redis (AOF persistence required).
- Anthropic Claude API can be used later to merge held items into a natural digest message.
- Vitest fake timers are the clearest way to verify digest-window behavior.
- Missed window recovery: if the machine was down when a digest window opened, held items should be processed at the next opportunity with staleness awareness — stale items are logged, not dispatched.
