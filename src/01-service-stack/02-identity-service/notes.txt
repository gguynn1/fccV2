Required tools/dependencies:
- Node.js (LTS) with TypeScript (strict, ESM) for all runtime modules.
- `src/types.ts` for shared enums and cross-service vocabulary.
- Respect service and type boundaries: no direct runtime imports from sibling services.
- Local `types.ts` in this folder for scope-owned contracts.
- Entity registry data for adults, child, and pet records.
- Permission and thread-membership lookups for routing and approval decisions.
- Stable sender-to-entity resolution before classification runs.

Recommended tools/dependencies:
- `tsc --noEmit` to verify type boundaries.
- `eslint` to keep imports and unused symbols clean.
- `prettier` to keep docs and source formatting consistent.
- `vitest` for fixture-driven verification where the scope has behavior worth testing.
- Fixtures covering private, couple, family, and pet-related lookups.
- Caching around repeated identity resolution if lookups become frequent.

Technologies and integrations:
- Identity resolution runs in strict TypeScript against entity and thread definitions owned by configuration/state.
- Validation is best enforced with Zod at config-load time so malformed entity records fail fast.
- Resolved identity data feeds classifier, routing, confirmation, and escalation decisions downstream.
