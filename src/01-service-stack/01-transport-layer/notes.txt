Required tools/dependencies:
- Node.js (LTS) with TypeScript (strict, ESM) for all runtime modules.
- `src/types.ts` for shared enums and cross-service vocabulary.
- Respect service and type boundaries: no direct runtime imports from sibling services.
- Local `types.ts` in this folder for scope-owned contracts.
- A phone-native messaging adapter interface for inbound and outbound traffic.
- Thread and participant mapping for the five allowed threads.
- Attachment, forwarded-content, and reaction normalization before items move downstream.

Recommended tools/dependencies:
- `tsc --noEmit` to verify type boundaries.
- `eslint` to keep imports and unused symbols clean.
- `prettier` to keep docs and source formatting consistent.
- `vitest` for fixture-driven verification where the scope has behavior worth testing.
- Webhook or event-fixture tests that cover text, reactions, images, and forwarded content.
- Retry and delivery-status instrumentation for outbound sends.

Technologies and integrations:
- Messaging platform: Twilio Programmable Messaging API for all phone-native inbound and outbound traffic.
- Inbound delivery: Twilio webhooks POST to the ngrok static subdomain, forwarded to Fastify on localhost:3000. Twilio request signature validation required on the webhook endpoint.
- Outbound delivery: Twilio REST API plus status callbacks for delivery updates. Retry with exponential backoff on network failure.
- Attachment handling: MMS media URLs downloaded and normalized before classification.
- Reaction handling: fallback to conversational reply parsing because native reactions are not guaranteed across the transport surface.
- Network outage: outbound sends queue locally via BullMQ retry until internet returns; inbound webhooks are unreachable while the tunnel is down — Twilio will retry delivery for a limited window.
