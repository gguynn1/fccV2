Required tools/dependencies:
- Node.js (LTS) with TypeScript (strict, ESM) for all runtime modules.
- `src/types.ts` for shared enums and cross-service vocabulary.
- Respect service and type boundaries: no direct runtime imports from sibling services.
- Local `types.ts` in this folder for scope-owned contracts.
- Access to the State service for reading calendar events across all topics.
- VCALENDAR/VEVENT formatting (ical.js or hand-rolled iCalendar output).

Recommended tools/dependencies:
- `tsc --noEmit` to verify type boundaries.
- `eslint` to keep imports and unused symbols clean.
- `prettier` to keep docs and source formatting consistent.
- `vitest` for fixture-driven verification where the scope has behavior worth testing.
- CalDAV compliance tests that verify PROPFIND, REPORT, and GET against real calendar apps.

Technologies and integrations:
- Served as Fastify routes on the same localhost:3000 server that handles Twilio webhooks.
- Exposed via the ngrok tunnel — same static subdomain, different path prefix (/caldav/*).
- Read-only: the endpoint serves VCALENDAR data but does not accept writes.
- Calendar events come from the State service — the CalDAV layer formats them, it does not own them.
- No external calendar API is consumed. The system is the authoritative calendar source.
