Required tools/dependencies:
- Node.js (LTS) with TypeScript (strict, ESM) for all runtime modules.
- `src/types.ts` for shared enums and cross-service vocabulary.
- Respect service and type boundaries: no direct runtime imports from sibling services.
- Routing-service outputs that distinguish response-in-place from proactive routing.
- Entity-to-thread mapping rules for private, couple, family, and pet cases.

Recommended tools/dependencies:
- `tsc --noEmit` to verify type boundaries.
- `eslint` to keep imports and unused symbols clean.
- `prettier` to keep docs and source formatting consistent.
- `vitest` for fixture-driven verification where the scope has behavior worth testing.
- Decision-table tests for narrowest-thread selection.
- Fixtures that separate immediate replies from later proactive follow-ups.

Technologies and integrations:
- This call depends on routing rules encoded in strict TypeScript over the shared thread map.
- Worker and routing service should exchange typed routing decisions, not ad hoc string targets.
- Vitest decision tables are the clearest technology fit for coverage here.
