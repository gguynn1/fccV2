Required tools/dependencies:
- Node.js (LTS) with TypeScript (strict, ESM) for all runtime modules.
- `src/types.ts` for shared enums and cross-service vocabulary.
- Respect service and type boundaries: no direct runtime imports from sibling services.
- Budget-service outputs for immediate, batched, and silent priority decisions.
- Collision-check data for recent sends, pending batched items, and quiet windows.

Recommended tools/dependencies:
- `tsc --noEmit` to verify type boundaries.
- `eslint` to keep imports and unused symbols clean.
- `prettier` to keep docs and source formatting consistent.
- `vitest` for fixture-driven verification where the scope has behavior worth testing.
- Time-based simulations that exercise digest windows and message spacing.
- Metrics around deferred, combined, and suppressed outbound items.

Technologies and integrations:
- Budget decisions rely on Redis-backed counters shared with BullMQ infrastructure.
- Quiet-window and batching behavior should be verified with Vitest fake-timer coverage.
- pino logs are useful for explaining why an outbound item was sent, held, or suppressed.
