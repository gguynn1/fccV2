Required tools/dependencies:
- Node.js (LTS) with TypeScript (strict, ESM) for all runtime modules.
- `src/types.ts` for shared enums and cross-service vocabulary.
- Respect service and type boundaries: no direct runtime imports from sibling services.
- Local `types.ts` in this folder for scope-owned contracts.
- Routing rules for same-thread responses and narrowest-thread proactive sends.
- Thread-map data covering private, couple, family, and pet-related cases.

Recommended tools/dependencies:
- `tsc --noEmit` to verify type boundaries.
- `eslint` to keep imports and unused symbols clean.
- `prettier` to keep docs and source formatting consistent.
- `vitest` for fixture-driven verification where the scope has behavior worth testing.
- Decision-table tests for audience selection.
- Fixtures for events that require both an immediate reply and a later proactive follow-up.

Technologies and integrations:
- Routing logic should stay in strict TypeScript over the shared thread map and entity types.
- Inputs come from worker classification/identity context and produce typed thread targets.
- Vitest decision-table coverage is the best way to verify routing behavior.
