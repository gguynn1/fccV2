Required tools/dependencies:
- Node.js (LTS) with TypeScript (strict, ESM) for all runtime modules.
- `src/types.ts` for shared enums and cross-service vocabulary.
- Respect service and type boundaries: no direct runtime imports from sibling services.
- Local `types.ts` in this folder for scope-owned contracts.
- Connector interfaces for inboxes, calendars, forwarded content, and future feeds.
- Extraction and pre-classification logic that normalizes external data into queue items.
- A queue producer for ingest-originated events.

Recommended tools/dependencies:
- `tsc --noEmit` to verify type boundaries.
- `eslint` to keep imports and unused symbols clean.
- `prettier` to keep docs and source formatting consistent.
- `vitest` for fixture-driven verification where the scope has behavior worth testing.
- Fixture corpora for emails, calendar changes, and forwarded images or messages.
- Parser validation for partial, noisy, or duplicate external inputs.

Technologies and integrations:
- Email monitoring: IMAP via imapflow for watched inboxes.
- Email extraction and image parsing should use Anthropic Claude API before queueing normalized items.
- Calendar input enters through email parsing or conversation, while the system serves a local CalDAV endpoint outward.
- New sources should produce queue items and must not require new paid third-party APIs without discussion.
- Network outage: IMAP connections drop when internet is down; imapflow should reconnect automatically on restore. Emails that arrived during the outage are processed on reconnection.
- Backlog processing: after downtime, the ingest service must process accumulated emails with staleness awareness — time-sensitive items that have passed their window should be stored silently, not dispatched as urgent alerts.
