Required tools/dependencies:
- Node.js (LTS) with TypeScript (strict, ESM) for all runtime modules.
- `src/types.ts` for shared enums and cross-service vocabulary.
- Respect service and type boundaries: no direct runtime imports from sibling services.
- Local `types.ts` in this folder for scope-owned contracts.
- A scheduler or timer abstraction for digests, reminders, follow-ups, and escalation deadlines.
- Queue item producer logic for scheduler-originated events.
- Daily-rhythm configuration for quiet windows and natural touchpoints.

Recommended tools/dependencies:
- `tsc --noEmit` to verify type boundaries.
- `eslint` to keep imports and unused symbols clean.
- `prettier` to keep docs and source formatting consistent.
- `vitest` for fixture-driven verification where the scope has behavior worth testing.
- A controllable clock for deterministic timer tests.
- Fixtures that simulate morning, daytime, and evening behavior.

Technologies and integrations:
- Scheduler jobs should run on BullMQ repeatable and delayed jobs backed by shared Redis (AOF persistence required).
- Timed events feed the same queue contract as every other source.
- Vitest fake timers are the right fit for digest-window and reminder timing tests.
- Missed window recovery: if the machine was down when a repeatable job should have fired (e.g., 7am morning digest), BullMQ fires it on next startup. The scheduler must check whether the window is still relevant — a morning digest at 2pm should be adapted or skipped, not sent as-is.
- Delayed job resilience: delayed jobs (reminders, escalation deadlines) survive restart if Redis AOF is enabled; without AOF they are lost.
