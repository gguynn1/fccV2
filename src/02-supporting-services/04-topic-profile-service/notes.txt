Required tools/dependencies:
- Node.js (LTS) with TypeScript (strict, ESM) for all runtime modules.
- `src/types.ts` for shared enums and cross-service vocabulary.
- Respect service and type boundaries: no direct runtime imports from sibling services.
- Local `types.ts` in this folder for scope-owned contracts.
- Topic profile definitions for tone, format, initiative style, escalation level, and any framework grounding.
- Stable topic-owned type contracts so each topic can evolve without leaking ownership elsewhere.

Recommended tools/dependencies:
- `tsc --noEmit` to verify type boundaries.
- `eslint` to keep imports and unused symbols clean.
- `prettier` to keep docs and source formatting consistent.
- `vitest` for fixture-driven verification where the scope has behavior worth testing.
- Per-topic fixtures that prove behavior differences stay intentional.
- A scenario matrix that checks profile outputs across all 12 topics.

Technologies and integrations:
- Topic profiles should live as strict TypeScript definitions keyed by shared `TopicKey` values.
- Zod is useful for validating profile configuration at boot.
- Vitest matrix coverage is the clearest way to verify all topics stay registered and distinct.
