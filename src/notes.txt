Required tools/dependencies:
- Node.js (LTS) with TypeScript (strict, ESM) for all runtime modules.
- `src/types.ts` for shared enums and cross-service vocabulary.
- Respect service and type boundaries: no direct runtime imports from sibling services.
- Local `types.ts` in this folder for scope-owned contracts.
- Root configuration and state stores such as `system-config.ts` and `system-state.ts`.
- Scenario-friendly bootstrapping so the whole system can start from a known state.

Recommended tools/dependencies:
- `tsc --noEmit` to verify type boundaries.
- `eslint` to keep imports and unused symbols clean.
- `prettier` to keep docs and source formatting consistent.
- `vitest` for fixture-driven verification where the scope has behavior worth testing.
- Scenario fixtures that exercise full end-to-end flows.
- Shared validation helpers for configuration and state stores.

Technologies and integrations:
- Runtime: Node.js (LTS) plus strict TypeScript running in ESM mode.
- Validation: Zod at configuration and state boundaries.
- Persistence: SQLite as the single state layer, accessed through the state service.
- Queueing and scheduling: BullMQ backed by shared Redis (AOF persistence required).
- Logging: pino for structured JSON logs.
- Testing: Vitest for scenario and fixture-based verification.
- Hosting: Mac Mini on a local network; three launchd-supervised processes (Node.js app, Redis, ngrok).
- Tunnel: ngrok with a static subdomain exposes the Fastify server for Twilio webhooks and CalDAV subscriptions.
- Resilience: machine sleep disabled, auto-restart after power failure enabled, stale catch-up logic on startup.
