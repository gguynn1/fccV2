Technologies and integrations (documented, not runtime):

- Batch aggregation: When the digest fires (BullMQ repeatable job), query all held items for the target person/thread and compose into one message.
- Message composition: Claude API merges multiple held items into a single natural digest message, shaped by each item's topic profile.
- Collision prevention: Budget service (Redis counters) ensures multiple messages are never stacked back to back.
